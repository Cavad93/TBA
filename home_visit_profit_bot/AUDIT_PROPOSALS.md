# Предложения на одобрение (Джаваду)

В код не идёт, пока не одобрено.

### Предложение — Бэкфилл координат старых «ручных» заказов

**Проблема:** заказы, созданные до сегодняшних правок с ручными км (без координат),
так и лежат без точки: не участвуют в автомаршруте и не показываются на карте.
**Что предлагаю:** разовый фоновый проход (или при открытии дня): прогнать адреса
таких визитов через новый слоёный геокодер и дозаполнить координаты.
**Зачем:** старые смены станут маршрутизируемыми, статистика км честнее.
**Цена вопроса:** маленький скрипт + квота DaData; риск — тихо изменившиеся
исторические цифры (нужно пометить пересчитанные).
**Статус:** ждёт одобрения.

### Предложение — Обратная синхронизация адресов дня сервер→телефон

**Проблема:** смена, начатая офлайн/до загрузки настроек, локально остаётся без
адресов старта/финиша, хотя сервер их знает («не задан» в Ленте до ручной правки).
**Что предлагаю:** в ответ `/api/route/active` включить адреса и координаты
старта/финиша дня, клиенту — дозаполнять локальный день, если там пусто.
**Зачем:** убирает последнее расхождение локальной и серверной картинки.
**Цена вопроса:** небольшое расширение API + миграция клиента; риск низкий.
**Статус:** ждёт одобрения.

### Предложение — Хранить на телефоне нормализованный адрес с сервера

**Проблема:** после правки якоря локально хранится введённый текст, на сервере —
нормализованный геокодером (координаты одни). Мелкое визуальное расхождение.
**Что предлагаю:** при успехе правки читать `start`/`finish` из ответа сервера и
класть в локальную БД нормализованную строку (и обновлять поле редактора, чтобы
сворачивание не залипло).
**Зачем:** одна строка адреса везде.
**Цена вопроса:** мелочь, но трогает UX сворачивания редактора — нужен аккуратный тест.
**Статус:** ждёт одобрения.

### Предложение — Перезапуск/починка codebase-memory MCP

**Проблема:** MCP-сервер графа отдаёт устаревший снапшот и не даёт пересоздать
проект («Permission denied» на delete). Устаревший граф уверенно отвечает
неправильно — это хуже, чем «не знаю».
**Что предлагаю:** перезапустить процесс MCP (вне сессии Claude), при повторе —
разобраться с блокировкой файла базы на Windows.
**Зачем:** без свежего графа правило «спрашивай граф, а не перечитывай файлы»
не работает.
**Цена вопроса:** перезапуск — минуты; починка блокировки — вопрос конфигурации.
**Статус:** ждёт одобрения (нужно действие вне сессии).

### Предложение — Показывать расходы «Машина»/«Аренда» в мастере завершения

**Проблема:** после починки записи (Этап 6) расходы категорий «Машина» и
«Аренда машины» реально копятся в дне, но мастер завершения смены их не
показывает в блоке «уже записанные расходы» — человек не видит полной картины
при подтверждении итогов.
**Что предлагаю:** добавить vehicle_expenses/vehicle_rent в expenses-словарь
предрасчёта и в клиентский экран мастера.
**Зачем:** полнота подтверждаемых итогов.
**Цена вопроса:** надо проверить, как клиент использует словарь expenses
(не просуммирует ли новые ключи дважды) — мелкая, но сквозная правка.
**Статус:** ждёт одобрения.

### Предложение — Цены лидов в матрице дня (офлайн-паритет)

**Проблема:** офлайн-оценка на телефоне строит before/after из кешированной
матрицы дня (`/api/route/matrix`), которая несёт доходы заказов, но не цены
откликов. После честного учёта лидов на сервере (Этап 4) офлайн-вердикт на днях
с платными лидами чуть оптимистичнее серверного.
**Что предлагаю:** добавить в ответ day_matrix массив `response_costs` в том же
порядке, что `incomes`, и учесть его в OfflineEstimateMapper.
**Зачем:** офлайн-вердикт перестаёт расходиться с серверным на платных лидах.
**Цена вопроса:** маленькое расширение API + клиент; риск низкий.
**Статус:** ждёт одобрения.

### Предложение — Подсказки адресов в форме «Начало дня»

**Проблема:** поля Старт/Финиш в «Начале дня» не проходят через слой подсказок:
опечатка уедет на сервер и день стартует без координат (адрес сохранится строкой).
**Что предлагаю:** тот же resolved/candidates-поток, что у якорей Ленты.
**Зачем:** координаты старта — фундамент всей оценки дня.
**Цена вопроса:** средняя правка UI; риск низкий.
**Статус:** ждёт одобрения.
