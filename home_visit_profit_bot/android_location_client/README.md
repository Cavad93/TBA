# Home Visit Android

Android-приложение для постепенного переноса функций Telegram-бота в отдельный мобильный интерфейс. Сейчас уже есть Material 3 каркас с вкладками и рабочая отправка GPS/агрегатов вождения.

## Что делает

- показывает основной shell приложения с вкладками `Сегодня`, `Работа`, `Маршрут`, `Отчеты`, `Усталость`, `Настройки`;
- работает только после нажатия кнопки `Начать отправку GPS`;
- запускает foreground service с постоянным уведомлением Android;
- отправляет `lat`, `lon`, `accuracy_m`, `provider`, `timestamp_ms` на `/location` примерно раз в минуту;
- собирает агрегаты акселерометра/гироскопа и GPS-скорости за день;
- отправляет на `/driving` только дневную сводку: резкие ускорения, торможения, повороты, прокси перестроений, старт-стоп, jerk, разброс скорости и общий индекс;
- не хранит постоянно и не отправляет сырые сенсорные события;
- содержит локальную Room-БД для рабочих дней, адресов, офиса, телемедицины, расходов, настроек и очереди синхронизации;
- уже имеет data-слой и ViewModel для локального сохранения действий;
- позволяет на вкладке `Работа` рассчитать домашний вызов через backend, увидеть решение и принять/отклонить адрес;
- показывает текущий принятый адрес и очередь адресов на вкладке `Маршрут`;
- позволяет завершить или отменить текущий адрес через backend;
- показывает GPS-подсказку текущего адреса через `/api/visits/current-gps` и позволяет закрыть адрес кнопкой `Закрыть по GPS`;
- позволяет на вкладке `Маршрут` уточнить тип GPS-остановки: `Обед/пауза`, `Ожидание`, `Обычный`, `Тяжёлый`, `Конфликтный`;
- позволяет начать день с вводом старта, финиша, сна, качества сна, перерыва и стартового одометра;
- позволяет быстро закрыть день с конечным одометром;
- позволяет закрыть день полной вечерней формой: рабочие километры, рабочие часы, время дороги, вызовы, конечный одометр, топливо, литры, расход л/100 км, компенсации, платные дороги, прочее и ручная усталость 0-100;
- умеет получить GPS-оценку активного дня с `/api/day/gps-estimate` и подставить в вечернюю форму рабочие часы, часы дороги и количество распознанных адресов;
- позволяет локально сохранить `ОФИС`, телемедицину и расход;
- отправляет локальную очередь изменений на backend через кнопку `Синхронизировать`;
- автоматически пробует синхронизировать очередь раз в 15 минут при наличии сети;
- для синхронизации использует endpoint `/api/sync`, для расчета адреса `/api/visits/candidate`, для GPS-подсказки `/api/visits/current-gps`, для уточнения остановки `/api/visits/{id}/stop-label`, а для GPS по-прежнему `/location` и `/driving`;
- не хранит адреса, ФИО пациентов и медицинские данные;
- список адресов и решение об уведомлении остаются на сервере бота.

## Вход в приложение

При первом запуске приложение просит зарегистрироваться (e-mail, ник, пароль) и
подтвердить e-mail кодом из письма. После входа выдаётся персональный токен сессии —
он используется как Bearer для всех запросов, поэтому вручную вводить API-ключ больше
не нужно. Данные каждого пользователя изолированы на сервере (PostgreSQL RLS).

Адрес сервера подставляется автоматически (`https://api.vizitorkrut.ru`). Выйти из
аккаунта или удалить его вместе со всеми данными можно на вкладке «Настройки».

## Настройки в приложении

- `URL сервера`: подставлен автоматически (`https://api.vizitorkrut.ru`), меняется при необходимости;
- `Интервал отправки`: минимум `60` секунд.

Если в поле URL указан адрес `/location`, приложение само преобразует его в `/api/sync` для отправки очереди.

Для точности в Android нужно выдать приложению разрешение на точную геолокацию.

## Сборка в GitHub

1. Загрузите весь проект `home_visit_profit_bot` в GitHub.
2. Откройте вкладку `Actions`.
3. Запустите workflow `Build Android APK`.
4. Скачайте артефакт `home-visit-android-debug-apk`.

Локальная сборка на Mac не обязательна. Основной способ сборки APK для этого проекта — GitHub Actions, где Gradle и Android SDK поднимаются автоматически.

Debug APK обычно нужно устанавливать вручную и разрешить установку из неизвестных источников.
