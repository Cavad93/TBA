# Визиторкрут — дизайн-система

Сама система лежит **не здесь**, а в корне репозитория:
[`Визиторкрут Design System/`](../../../Визиторкрут%20Design%20System/)

Этот файл — только указатель. Так сделано намеренно: система (95 файлов, ~650 КБ,
включая сгенерированные `_ds_bundle.js` и `_ds_manifest.json`) — выгрузка из
дизайн-инструмента. Копия в `.claude/skills/` разошлась бы с оригиналом при
следующем экспорте, а два источника правды по бренду хуже, чем один с лишним шагом.

## Правила бренда — не нарушать

- **Зелёный = бренд и «стоит ехать».** Спайн вердикта (зелёный / янтарный / красный)
  священен: этими цветами нельзя красить декор.
- **Числа ведут макет**, набираются табличным моноширинным.
- К работнику обращаемся **на «ты»**. Эмодзи — нет.
- Тёплая бумага + тёплые чернила, чёткие границы 1px, крупные радиусы,
  резкая анимация без отскока.
- **Логотипа не существует.** Имя «Визиторкрут» набирается шрифтом Unbounded;
  выдумывать знак нельзя.

## Продакшн против макетов

- **Продакшн-код** (сайт `home_visit_profit_bot/deploy/site/`): сайт — статика,
  React там нет. Читаем токены и `.prompt.md`, применяем **значения** в `ds.css`
  и разметке. Компоненты `.jsx` — источник знаний об API, а не код для вставки.
- **Макеты и прототипы**: статический HTML, который можно открыть. Подключить
  `styles.css`, React + Lucide UMD + `_ds_bundle.js`, компоненты монтируются из
  `window.DesignSystem_ee81bc`. Порядок загрузки — подсмотреть в любом
  `ui_kits/*/index.html`.

## `_adherence.oxlintrc.json` — не работает, не тратить на него время

В системе лежит `_adherence.oxlintrc.json`, похожий на конфиг oxlint для проверки
соответствия DS. **Запустить его нельзя** — проверено 17.07.2026:

- поле `x-omelette` в корне ломает разбор конфига (oxlint такого не знает);
- если его убрать — падает на `no-restricted-syntax`: в текущем oxlint нет
  такого правила;
- и даже заработай он — правила завязаны на плагины `react`/`import` и
  ограничение импортов из `components/**`, то есть он про **React-код**.
  Сайт — статический HTML+CSS, импортов там нет вообще.

Похоже, это артефакт стороннего инструмента (отсюда и `x-omelette`), а не
рабочая проверка. **Соответствие DS проверяется чтением токенов и глазами.**
Если понадобится машинная проверка — её надо писать заново под CSS, а не
чинить этот файл.
