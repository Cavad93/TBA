# Граф кода «Визиторкрута»

Индекс всего кода в виде графа: функции, классы, файлы и рёбра между ними
(`CALLS`, `IMPORTS`, `DEFINES`, `INHERITS`). Отвечает на структурные вопросы за
миллисекунды и ~500 токенов там, где `grep` + чтение файлов сжигают десятки тысяч.

Инструмент — `codebase-memory-mcp` (MCP-сервер, ставится в `~/.local/bin`, граф
живёт в `~/.cache/codebase-memory-mcp/`, в репозиторий ничего не пишет).

## Два проекта, а не один. Это принципиально

| Что | Имя проекта в графе | Корень |
|---|---|---|
| Бэкенд (Python) | `…-Desktop-TBA-home_visit_profit_bot` | `home_visit_profit_bot/` |
| Android (Kotlin) | `…-Desktop-TBA-home_visit_profit_bot-android_location_client` | `home_visit_profit_bot/android_location_client/` |

Точные имена — `list_projects`. Они выводятся из абсолютного пути, поэтому на другой
машине будут другими.

**Не индексировать корень репозитория `TBA/` одним проектом.** Так было сделано в
первый раз, и получилось два разных отравления графа — оба молчаливых:

**Ловушка 1. Корень индексации обязан быть корнем импорта.**
Импорты в бэкенде выглядят как `from app.services.parking_service import find_zone`,
а `app/` лежит внутри `home_visit_profit_bot/`. Проиндексируешь от `TBA/` — индексатор
ищет `app/…` от корня, не находит и **молча выбрасывает ребро вызова**. Проверено:
`find_zone` и `get_distance_matrix` показывали **ноль вызывающих**, хотя их зовут
`visit_parking.py:31` и `optimization_service.py:35`. Вызовы методов (`obj.method()`)
при этом сохранялись — то есть граф выглядел рабочим и врал выборочно.

**Ловушка 2. Один язык — один граф.**
Пока Kotlin и Python лежали в одном графе, индексатор связывал вызовы **по совпадению
имени** и придумывал рёбра между языками: он всерьёз утверждал, что Compose-компонент
`AppSettingsCard` вызывает питоновский `profile_service.snapshot()`. Разные языки,
разные процессы, никакого вызова между ними нет и быть не может.
Лечится файлом `home_visit_profit_bot/.cbmignore` — он исключает Android-клиент из
питоновского графа. Android индексируется отдельным проектом.

## Какой вопрос — какой инструмент

| Вопрос | Вызов |
|---|---|
| Кто вызывает X? | `trace_path(function_name="X", direction="inbound", depth=2)` |
| Что вызывает X? | `trace_path(..., direction="outbound")` |
| Найти по имени / шаблону | `search_graph(name_pattern=".*Parking.*")` |
| Найти по смыслу | `semantic_query` |
| Обзор архитектуры | `get_architecture(aspects=["all"])` |
| На что повлияет моя правка | `detect_changes()` — берёт git diff и показывает задетые символы |
| Мёртвый код | `search_graph(label="Function", max_degree=0, exclude_entry_points=true)` |
| Произвольный обход | `query_graph` (Cypher: `MATCH (a)-[:CALLS]->(b) WHERE …`) |
| Прочитать исходник символа | `get_code_snippet(qualified_name="…")` |

## Чего в графе НЕТ

- `home_visit_profit_bot/deploy/` — индексатор исключает папку `deploy` своим встроенным
  правилом. Значит `Caddyfile`, `Dockerfile`, сайт и `migrate_sqlite_to_postgres.py`
  графу не видны совсем. Про них — только `grep`.
- `Визиторкрут Design System/`, `.venv/`, `__pycache__/`.
- Всё, что на серверах (OSRM, скрипты `/opt/osrm/*.sh`) — это не репозиторий.

## Главное правило: граф — это карта, а не территория

**«Нет вызывающих» ≠ «мёртвый код».** В графе есть только вызовы из кода. Всё, что
дёргает фреймворк, вызывающих не имеет и выглядит мёртвым:

- тесты — их собирает pytest по имени (граф честно записал два наших теста в мёртвый код);
- обработчики HTTP — их зовёт таблица маршрутов в `location_api.py`;
- Compose-компоненты — их зовёт рантайм.

Поэтому: **перед тем как что-то удалить или отрефакторить по совету графа — сверить
грепом.** Граф хорош, чтобы быстро найти, куда смотреть; решение принимается по коду.

Это ровно та же дисциплина, что и с OSRM: инструмент, который уверенно отвечает
неправильно, опаснее инструмента, который честно отвечает «не знаю».

## Переиндексация

Фоновый наблюдатель подхватывает правки сам. Руками — когда сменилась структура
(добавили пакет, тронули `.cbmignore`):

```
codebase-memory-mcp cli index_repository '{"repo_path": "…/home_visit_profit_bot"}'
codebase-memory-mcp cli index_repository '{"repo_path": "…/home_visit_profit_bot/android_location_client"}'
```

Инкрементальная переиндексация **не перечитывает `.cbmignore`**. Меняли его — снести
проект (`delete_project`) и построить заново, иначе изменения не применятся и вы этого
не заметите.
