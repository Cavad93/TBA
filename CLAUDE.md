# Правила работы над проектом «Визиторкрут»

Эти правила обязательны для соблюдения при работе над репозиторием.

## Как работать

1. **Язык общения — всегда русский.** Все ответы, пояснения, сообщения коммитов и
   комментарии для пользователя пишутся на русском языке.

2. **Выгружать в GitHub после каждого этапа.** По завершении каждого логического
   этапа (фазы, задачи) — коммит и `git push` в удалённый репозиторий, чтобы
   прогресс не терялся и запускалась сборка (например, APK через GitHub Actions).

3. **Спрашивать, если что-то не до конца понятно.** Не додумывать и не угадывать
   в неоднозначных ситуациях — уточнять у пользователя до начала реализации.

4. **Предлагать идеи.** Если есть улучшения, альтернативы или полезные замечания —
   активно их предлагать, а не только выполнять буквально.

5. **Проверять чувствительные места реализации в сети.** Ответственные и рискованные
   решения (безопасность, платежи, персональные данные / 152-ФЗ, юридические
   моменты, редкие API и их актуальные сигнатуры) сверять с актуальными источниками
   в интернете перед внедрением.

## Контекст проекта

- Продукт: «Визиторкрут» (`vizitorkrut.ru`) — массовое многопользовательское
  приложение для выездных работников (врачи, таксисты, курьеры, мастера на дом).
- Девиз: «Навигатор показывает, куда ехать. Наше приложение показывает — стоит ли ехать».
- Бэкенд + сайт: Ubuntu VPS, PostgreSQL с изоляцией данных по пользователю (RLS),
  Caddy (авто-HTTPS), корпоративная почта Yandex 360.
- Android-клиент: Kotlin/Compose, сборка APK через GitHub Actions.
